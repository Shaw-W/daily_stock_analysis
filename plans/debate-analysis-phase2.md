# 第二阶段计划：多智能体辩论与现有分析体验深化

## 目标

在第一阶段独立辩论接口稳定后，增强 Web 展示、历史查询和普通分析联动，但仍保持普通分析默认快速稳定。

## 重点任务

1. 历史查询优化：为 `report_type=debate` 增加筛选、列表标签和详情页专用展示。
2. Web 辩论详情页：将六 Agent、多方、空方、裁判拆成可折叠区块，增加投票概览和胜率卡片。
3. 普通分析联动：在普通分析报告详情中提供“发起辩论”按钮，复用当前股票与历史上下文。
4. 报告对比：支持同一股票普通分析与辩论结论并列查看，标注分歧点。
5. 通知增强：按渠道选择更合适模板，进一步优化 Telegram/飞书展示。
6. 测试覆盖：补齐 API、任务队列、历史详情和前端页面回归测试。

## 不做

- 不默认把辩论合入普通分析报告。
- 不开放复杂 Agent 策略自定义。
